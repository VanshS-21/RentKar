# RentKar - Team Roles & Responsibilities

## Team Structure (3 Members)

---

## Member 1: Backend Lead

### Primary Responsibilities
- Spring Boot project setup and configuration
- Database schema design and implementation
- User authentication and JWT security
- Core business logic development
- API endpoint development
- AI service integration

### Weekly Tasks Breakdown

**Week 1-2**: Setup & Authentication
- Initialize Spring Boot project with dependencies
- Configure MySQL database connection
- Create User entity and repository
- Implement JWT authentication service
- Build login/register endpoints
- Write unit tests for auth service

**Week 3-4**: Item Management Backend
- Create Item entity and repository
- Implement Item service layer
- Build CRUD endpoints for items
- Add validation and error handling
- Integrate Cloudinary for image upload

**Week 5**: AI Integration
- Research and select LLM API
- Create AI service wrapper
- Build description generation endpoint
- Test and optimize AI prompts

**Week 6-7**: Borrow Request Backend
- Create BorrowRequest entity
- Implement request service logic
- Build request management endpoints
- Handle status transitions

**Week 8-11**: Testing & Optimization
- Write comprehensive unit tests
- API integration testing
- Performance optimization
- Bug fixes

---

## Member 2: Frontend Lead

### Primary Responsibilities
- React project setup with Vite
- UI component development
- State management implementation
- API integration with backend
- Responsive design with TailwindCSS
- Form handling and validation

### Weekly Tasks Breakdown

**Week 1-2**: Setup & Authentication UI
- Initialize React project with Vite
- Setup TailwindCSS configuration
- Create routing structure
- Build Login/Register pages
- Implement form validation
- Connect auth APIs

**Week 3-4**: Item Management UI
- Create item listing page
- Build item detail page
- Implement item creation form
- Add image upload component
- Create item card components

**Week 5**: AI Integration UI
- Add "Generate with AI" button
- Handle AI loading states
- Display generated content
- Allow editing of AI suggestions

**Week 6-7**: Borrow Workflow UI
- Create request button component
- Build lender dashboard
- Implement request approval UI
- Add return confirmation flow
- Create transaction history view

**Week 8-11**: UI Polish & Testing
- Responsive design refinement
- Loading states and error handling
- Toast notifications
- Component testing
- Bug fixes

---

## Member 3: Full-stack / Integration Specialist

### Primary Responsibilities
- Frontend-Backend integration
- Cloudinary setup and integration
- Borrow workflow implementation
- End-to-end testing
- Deployment preparation
- Documentation

### Weekly Tasks Breakdown

**Week 1-2**: Project Setup Support
- Setup Git repository
- Create project documentation structure
- Help with environment setup
- Create API contract document

**Week 3-4**: Image Upload Integration
- Setup Cloudinary account
- Integrate Cloudinary SDK (backend)
- Implement image upload flow
- Test image upload end-to-end

**Week 5**: AI Feature Testing
- Test AI integration
- Optimize prompts
- Handle edge cases
- Document AI usage

**Week 6-7**: Borrow Workflow Integration
- Connect request APIs with UI
- Test complete borrow flow
- Implement status updates
- Handle error scenarios

**Week 8**: Optional Features
- Admin dashboard (if time permits)
- Basic messaging system
- Search and filter functionality

**Week 9-10**: Testing & Bug Fixes
- End-to-end testing
- Cross-browser testing
- Mobile responsiveness testing
- Bug tracking and fixing

**Week 11-12**: Documentation & Deployment
- Write technical documentation
- Create API documentation
- Prepare deployment scripts
- Create demo video
- Prepare presentation

---

### All Team Members
- Daily standup meetings (15 min)
- Code reviews for pull requests
- Weekly progress updates
- Documentation updates
- Testing their own features
- Bug fixing in their domain

### Communication Channels
- **Daily Updates**: WhatsApp/Slack group

## Weekly Meeting Agenda Template

### Monday Standup (15 min)
- What did you complete last week?
- What will you work on this week?
- Any blockers or help needed?

### Friday Review (30 min)
- Demo completed features
- Code review session
- Plan next week's tasks
- Update project board

---

## Conflict Resolution
- Technical disagreements: Team discussion + voting
- Task overload: Redistribute tasks
- Blockers: Immediate team huddle
- Missed deadlines: Adjust timeline collectively

---

### Member 2 (Frontend)
